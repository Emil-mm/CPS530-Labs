Part C - Comparison Analysis

It took me much longer to plan the grid and figure out how to change the format of the responsive design of the webpage in part A. In part A, when trying to make the div containers of equal height using float some of the sections on my page started to disappear for an unknown reason. Formating the div blocks was difficult, I had to learn how to stack and control them to create the desired format at specific sizes with @media.

In part B, Bootstrap took me less time to become familiar with using the grid system with the simple use of the "col-class-span" classes. It was ultimately much easier to format the webpage from mobile to tablet to PC responsive formatting. When using the grid "col-class-span" classes it automatically made the div containers in the same row equal height and changed the screen without an explicitly specified screen size value. However, when I was trying to make my image fill the whole div container, it took me a while to realize there was a default padding included in the "col-class-span" class>. The only challenge I found was that we could not see the properties used in the classes. I prefer part B due to the cleaner code and simpler framework.
